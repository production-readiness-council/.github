# Security Policy

## Scope

This policy covers vulnerabilities in software artifacts and operational workflows maintained by the Production Readiness Council.

## Reporting

Do not disclose unpatched vulnerabilities publicly first.

Use a private reporting channel designated by the organization owners or maintainers. If no private channel is documented yet, open a private GitHub security advisory if repository settings allow it, or contact an organization owner directly.

## Handling expectations

Security reports should include:

- affected repository or workflow
- impact summary
- reproduction details when safe to share
- suggested mitigation if known

## Disclosure

Public disclosure should wait until the issue has been reviewed and an agreed handling path exists.

## No guarantees

Response times and remediation windows are not guaranteed and should not be implied unless the organization publishes them explicitly.

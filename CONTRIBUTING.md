# Contributing

## Principles

Contributions should improve technical quality, governance clarity, evidence quality, reproducibility, or operational simplicity.

## Default workflow

1. Open or join a relevant issue.
2. State the claim, question, or proposed change clearly.
3. Link evidence, prior discussion, or affected governance artifact where possible.
4. Keep pull requests scoped to one decision or change set.
5. Expect review for correctness, governance impact, and traceability.

## Contribution standards

- Use neutral language.
- Mark drafts clearly.
- Do not imply approval, publication, or endorsement before the documented gate is satisfied.
- Do not introduce unsupported metrics, affiliations, or credentials.
- Preserve public traceability for decisions that affect governance or report workflow.

## Governance changes

Changes to governance artifacts should:

- explain the current rule
- explain the proposed change
- explain the expected workflow impact
- identify which role or body has approval authority

## Security-sensitive content

Do not post secrets, live credentials, or exploitable vulnerability details in public issues or pull requests.
